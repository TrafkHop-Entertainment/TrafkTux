Erklärung der wichtigsten Änderungen

    run_menu_session() ersetzt das frühere main() und steuert den gesamten Fluss – ohne os.execv.

    Navigation (Ordner, Seiten) aktualisiert nur die Variablen path_str und page und springt zum Schleifenanfang.

    Virtuelle Menüs (special‑drun/special‑run) werden in run_virtual_menu() abgewickelt; bei Zurück wird einfach "back" zurückgegeben, woraufhin die Hauptschleife den vorherigen Zustand beibehält.

    Daemon‑Schleife wartet in einer blockierenden read() auf der FIFO – sobald show kommt, wird das Menü gestartet. Nach Beendigung (App‑Start oder Escape) kehrt der Daemon in den Wartezustand zurück.

    Exit im Menü löst ein "exit" aus, das den Daemon sauber beendet (inkl. Löschen der FIFO und der temporären Fensterdatei).

Systemstart (Hyprland)

Füge in deine Hyprland‑Konfiguration (~/.config/hypr/hyprland.conf) ein:
conf

exec-once = /pfad/zu/bubble-menu.py --menu launcher --daemon

und für einen Hotkey:

Für das Powermenü starte einen zweiten Daemon mit --menu powermenu und einem anderen Hotkey.
Hinweise

    Die FIFO wird mit 0o600 erstellt – nur der eigene Benutzer kann lesen/schreiben.

    Der Daemon reagiert auf SIGTERM und räumt auf.

    Alle Importe bleiben beim ersten Start einmalig, die Ladezeit reduziert sich auf den ersten Aufruf – alle weiteren Menüöffnungen sind sofort da.

    Der Daemon läuft dauerhaft im Hintergrund und verbraucht kaum Ressourcen (blockierendes read auf der FIFO).

Powermenü im Daemon‑Betrieb

Im bisherigen Code wird der Daemon mit einem festen FIFO‑Pfad (/tmp/rofi‑bubble.fifo) gestartet.
Wenn du sowohl das Hauptmenü (launcher) als auch das Powermenü als Daemon bereitstellen möchtest, brauchst du zwei separate FIFOs – sonst überschreiben sich die Befehle.
1. FIFO‑Pfad anpassen (empfohlen)

Ändere die Konstante FIFO_PATH so, dass sie den Menü‑Namen enthält:
python

FIFO_PATH = f"/tmp/rofi-bubble-{menu_name}.fifo"

Damit bekommst du z. B.

    /tmp/rofi-bubble-launcher.fifo

    /tmp/rofi-bubble-powermenu.fifo

2. Zwei Daemons starten (z. B. in hyprland.conf)
conf

exec-once = /pfad/zu/bubble-menu.py --menu launcher --daemon
exec-once = /pfad/zu/bubble-menu.py --menu powermenu --daemon

3. Hotkeys für beide Menüs
conf

bind = $mainMod, Space, exec, echo "show" > /tmp/rofi-bubble-launcher.fifo
bind = $mainMod, Shift+Space, exec, echo "show" > /tmp/rofi-bubble-powermenu.fifo

4. Alternative: Ein Daemon für mehrere Menüs

Falls du lieber einen einzigen Daemon betreiben möchtest, der auf Befehle wie show:launcher oder show:powermenu reagiert, müsstest du die FIFO‑Nachricht parsen und dann das entsprechende Menü starten. Das ist aber aufwändiger und bringt kaum Vorteile – zwei separate Daemons sind einfacher und genauso ressourcenschonend.
5. Wichtig: FIFO‑Pfad im Code anpassen

Füge in der daemon_loop()‑Funktion die Verwendung des Menü‑Namens ein:
python

def daemon_loop(menu_name: str, x11: bool):
    fifo_path = f"/tmp/rofi-bubble-{menu_name}.fifo"
    # ... Rest wie gehabt, aber überall fifo_path statt FIFO_PATH verwenden

6. Starten des Powermenüs im Nicht‑Daemon‑Modus

Wenn du das Skript ohne --daemon aufrufst (einmalig), bleibt alles wie gehabt:
bash

bubble-menu.py --menu powermenu

Das öffnet sofort das Powermenü und beendet sich nach der Auswahl.
Im Daemon‑Modus dagegen bleibt der Prozess erhalten und reagiert auf das show‑Signal – das ist genau das, was du wolltest: keine Neuinitialisierung mehr, sondern sofortige Anzeige.
Kurz zusammengefasst

    Powermenü wird genau wie das Hauptmenü als eigener Daemon gestartet.

    Die FIFOs werden pro Menü unterschiedlich benannt, damit sie sich nicht in die Quere kommen.

    Die Hotkeys senden dann einfach show an die entsprechende FIFO.

Damit hast du beide Menüs persistent und verzögerungsfrei verfügbar.
